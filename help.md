# 📖 NPlace-DB 마케팅 엔진 최종 매뉴얼

본 가이드는 NPlace-DB의 안정적인 운영과 문제 해결을 위한 지침서입니다.

---

### **1. 🧹 발송 결과 초기화 (Reset)**
*   **기능**: 이미 발송된 결과(Success/Fail)와 로그를 모두 지우고 처음부터 다시 발송하고 싶을 때 사용합니다.
*   **위치**: 대시보드 화면 최하단의 **[🧹 발송 결과 전체 초기화]** 버튼.
*   **효과**: 데이터베이스의 모든 발송 기록이 삭제되며, 상단의 성공/실패 카운트도 0으로 초기화됩니다.

### **2. ⏯️ 일시정지 및 재개 (Smart Resume)**
*   **일시정지**: 별도의 버튼 없이 프로그램 창을 닫거나 콘솔 창을 종료하면 즉시 중단됩니다.
*   **재개(Resume)**: 다시 실행하면 **[스마트 리트라이]** 기능이 자동으로 작동합니다. 이미 성공(`Success`)으로 표시된 업체는 자동으로 건너뛰고, 발송하지 않았거나 실패한 업체들만 골라서 발송을 이어갑니다. 
*   **주의**: 중단 전까지 성공한 데이터는 실시간으로 DB에 저장되어 있으니 안심하세요.

### **3. 🔍 결과 확인 및 상세 로그 (Details)**
*   **결과(E/I)**: 발송 성공 여부 (`Success` / `Fail`).
*   **로그(E/I)**: 
    *   `발송완료`: 정상적으로 전송됨.
    *   `에러 메시지`: SMTP 인증 오류, 네트워크 문제, 잘못된 URL 등 구체적인 실패 사유를 확인할 수 있습니다.

### **4. 🛠️ 문제 해결 (Troubleshooting)**
*   **화면 갱신 문제**: 발송 직후 결과가 바로 보이지 않는다면 상단의 **[🔄 데이터 갱신]** 버튼을 눌러주세요.
*   **DB 잠김 오류**: 간혹 "Database is locked" 에러가 발생하면, 프로그램을 끄고 5초 뒤에 다시 실행하세요.
*   **초기화 에러**: 초기화 버튼 클릭 시 에러가 발생하면 프로그램을 최신 버전으로 업데이트하거나 완전히 재시작하세요.

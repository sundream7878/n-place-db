# 📘 NPlace-DB 공식 사용자 가이드 & 문제 해결 매뉴얼

**최종 업데이트 일시: 2026-05-06 15:01**
**버전: V1.1.0 (Production Stable)**

본 문서는 NPlace-DB 사용자를 위한 통합 매뉴얼입니다. 장애 발생 시 아래의 순서에 따라 조치해 주시기 바랍니다.

---

## 1. 🧹 발송 결과 및 데이터 초기화 (Reset)
NPlace-DB는 동일 업체 중복 발송을 방지하기 위해 모든 결과를 데이터베이스(DB)에 기록합니다.

### **[결과 초기화가 필요한 경우]**
- 모든 업체에 대해 처음부터 다시 발송하고 싶을 때
- 테스트 데이터를 모두 지우고 실제 작업을 시작할 때

### **[조치 방법]**
1.  대시보드 메인 화면 최하단으로 스크롤합니다.
2.  `🛠️ 데이터 및 발송 결과 관리` 섹션을 클릭하여 확장합니다.
3.  **`🧹 발송 결과 전체 초기화`** 버튼을 누릅니다.
4.  "데이터가 초기화되었습니다" 메시지가 뜨면, 이후 실행 시 모든 업체에 대해 다시 발송을 시작합니다.

---

## ⏸️ 2. 작업 일시정지 및 중단 (Pause & Stop)
발송 도중 작업을 멈추고 싶을 때의 올바른 절차입니다.

### **[정상 중단 절차]**
1.  대시보드 상단의 **`작업 중단`** 버튼을 클릭합니다.
2.  현재 진행 중인 1건의 업체까지만 발송을 마친 후 안전하게 멈춥니다.

### **[강제 중단 절차]**
- 프로그램이 응답하지 않을 경우: 런처 창을 닫거나 작업 관리자(`Ctrl+Shift+Esc`)에서 `NPlace-DB.exe`를 종료합니다.
- **다시 시작할 때**: 별도의 초기화 없이 프로그램을 다시 켜면, **이미 성공한 업체는 건너뛰고** 마지막으로 멈췄던 업체부터 자동으로 이어 발송합니다 (스마트 리트라이).

## ⚠️ 3. 상세 문제 발생 시 조치 사항 (Troubleshooting)

### **🚩 상황 1: 시작 로고(Splash) 화면이 사라지지 않고 계속 떠 있음**
- **현상**: 메인 화면이 떴는데도 가운데 로고 이미지가 사라지지 않고 다른 창을 가리는 경우.
- **원인**: 윈도우 시스템의 리소스 점유 상태에 따라 로고 종료 명령이 지연된 상태입니다.
- **해결책**:
    1.  가운데 떠 있는 **로고 이미지 창을 한 번 클릭**합니다.
    2.  키보드에서 **`Alt + F4`** 단축키를 눌러 해당 창만 강제 종료합니다.
    3.  메인 프로그램은 정상 작동 중이므로 그대로 사용하시면 됩니다.

### **🚩 상황 2: 화면 양옆에 넓은 여백(흰 공간)이 생기는 경우**
- **현상**: 대시보드 내용이 가운대로 몰려 보이고 좌우가 텅 빈 경우.
- **해결책**:
    - 프로그램 우측 상단의 `...` (설정) 메뉴에서 `Wide mode`가 활성화되어 있는지 확인하십시오.
    - V1.1.0부터는 **풀-와이드(Full-Width) CSS**가 적용되어 있어, 새로고침(F5)을 하면 화면 전체로 확장됩니다.

### **🚩 상황 3: "연결 거부" 또는 "페이지를 찾을 수 없음"**
- **현상**: 브라우저 창에 하얀 화면만 뜨거나 연결할 수 없다고 나오는 경우.
- **해결책**:
    - 런처 실행 후 대시보드 서버가 준비되는 데 약 5~10초가 소요됩니다.
    - 조금만 기다렸다가 **`Ctrl + R` 또는 `F5`**를 눌러 페이지를 새로고침 하십시오.

## 🧠 4. NPlace-DB만의 스마트 리트라이 시스템
본 프로그램은 단순한 반복 작업이 아닌 **지능형 추적 시스템**을 탑재하고 있습니다.

1.  **ID 기반 매칭**: 업체명이나 URL이 바뀌어도 고유한 **Shop ID**를 통해 중복 여부를 판별하므로 누락이나 중복 발송이 없습니다.
2.  **실시간 상태 반영**: 성공하면 즉시 `성공`으로 기록되며, 프로그램이 비정상 종료되어도 다시 켰을 때 **성공한 리스트는 알아서 제외**하고 나머지 작업을 이어갑니다.

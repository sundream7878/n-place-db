# 🚨 N-Place-DB 수집기 엔진 무한 대기(Hang) 이슈 리포트

## 1. 개요
현재 N-Place-DB 대시보드에서 '데이터 수집 시작'을 클릭했을 때, 수집기 엔진(`step1_refined_crawler.py`)이 브라우저를 띄우기 전 단계에서 무한 대기(Hang) 현상이 발생하고 있습니다. 사용자가 약 7회 이상의 세션 수정을 거쳤으나 로컬 환경에 따라 간헐적 혹은 지속적으로 엔진이 멈추는 현상이 해결되지 않고 있습니다.

## 2. 상세 증상
- **로그 정지 지점**: `2026-04-28 14:48:30,472 - INFO - 🧹 Cleaning up old browser/driver processes...`
- **현상**: 위 로그 출력 이후 다음 단계(Launching System Chrome...)로 넘어가지 못하며, N플레이스 창(Playwright 브라우저)이 나타나지 않음.
- **결과**: 대시보드 UI는 '엔진 준비 중' 상태에 머물러 있으며, 데이터 수집이 전혀 진행되지 않음.

## 3. 기술적 원인 분석 (추정)
1. **좀비 프로세스 락(Lock)**: 이전 크롤링 세션이 비정상 종료되거나 강제 종료되었을 때, Playwright의 백엔드 드라이버인 `node.exe`와 `chrome.exe`가 메모리에 남아서 다음 세션의 브라우저 기동을 방해함.
2. **동기식 명령어 교착(Deadlock)**: `os.system("taskkill ...")` 호출 시 윈도우 시스템 환경에 따라 명령어가 즉시 반환되지 않고 대기 상태에 빠질 수 있음. 이로 인해 메인 루프가 차단됨.
3. **Playwright Driver 타임아웃**: 드라이버가 초기화되는 과정에서 로컬 파일(Temp 디렉토리 등)에 접근 권한 문제나 파일 락이 걸려 있을 경우, 별도의 에러 메시지 없이 영구 대기할 수 있음.

## 4. 지금까지 시도한 조치 (Summary)
- **로깅 시스템 개선**: `FileHandler` 인코딩 설정 및 강제 플러시(`flush()`)를 통해 대시보드에서 실시간 로그 확인이 가능하도록 조치함.
- **예상 수치 고정**: 체험판 등 한도 설정 시 `estimated_total`이 유동적으로 변하지 않고 목표 수치에 고정되도록 수정함.
- **중복 스킵 알림**: 이미 수집된 데이터를 스킵할 때도 대시보드 UI에 진행률 신호를 보내어 멈춘 것처럼 보이지 않게 조치함.
- **사전 청소 로직**: `async_playwright()` 호출 직전 `taskkill` 명령을 통해 좀비 프로세스를 제거하려 시도함.

## 5. 다음 작업자를 위한 제안 (Next Actions)
1. **비동기 프로세스 처리**: `os.system` 대신 `asyncio.create_subprocess_exec` 등을 사용하여 타임아웃이 적용된 비동기 프로세스 정리를 구현해야 함.
2. **드라이버 독립 실행**: Playwright 드라이버가 꼬였을 경우를 대비해, `%LOCALAPPDATA%\ms-playwright` 경로의 임시 파일이나 락 파일을 수동으로 체크하고 제거하는 로직 검토 필요.
3. **디버그 모드 강화**: `DEBUG` 레벨 로그를 켜서 Playwright 내부의 통신 로그(`DEBUG=pw:browser`)를 파일로 남겨 정확한 프리징 지점을 파악해야 함.

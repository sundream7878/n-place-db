## 2. 해결이 필요한 잔여 이슈 (Next Session Tasks)

### A. 메인 대시보드 로그창 스크롤바 문제
- **증상**: [수집 상세 현황] 하단의 실시간 엔진 로그창에서 스크롤바가 로그 박스 우측에 붙지 않고, 전체 대시보드 화면 끝에 생성되는 현상 발생.
- **원인 추정**: Streamlit의 Container 구조 내에서 `log-container` CSS의 `height` 및 `overflow` 속성이 브라우저 레이아웃에 의해 무시되거나 부모 요소로 전이됨.
- **요구사항**: 로그 박스 내부에 전용 스크롤바가 확실히 고정되어야 하며, 박스 높이가 고정되어야 함.

### B. 이메일 발송 로그인 실패 (535 Error)
- **증상**: 네이버 계정(`chiu369@naver.com`)으로 발송 시 `(535, b'5.7.1 Username and Password not accepted')` 에러 지속.
- **분석**:
    1. **비밀번호 확인**: 관리창에서 확인 시 비밀번호 끝에 `)` 문자가 포함되어 있을 가능성 농후 (사용자 입력 실수 확인 필요).
    2. **환경설정**: 네이버 메일의 [POP3/IMAP 설정]이 '사용함'인지 재확인 필수.
    3. **보안 인증**: 2단계 인증 사용 시 '앱 암호' 사용 필수.
- **조치**: 발송 엔진은 현재 SSL(465) 및 STARTTLS(587)를 모두 지원하도록 업데이트됨.

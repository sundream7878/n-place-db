# 제목: N-Place-DB Pro 멀린 개발 협업 강령 (Merlin's Code of Process)
- **버전**: v1.0.0
- **일시**: 2026-05-18 16:38

---

## 📜 서문 (Introduction)
본 **멀린 협업 강령(Merlin's Process Creed)**은 대표님과 멀린(AI 어시스턴트)이 세션이 바뀌고 맥락이 리셋되더라도 항상 최상의 결과물과 안전성을 보장하기 위해 지켜야 할 **프로세스적 협업 원칙**을 정의한다.

---

## 🧭 핵심 강령 규칙 (Core Rules)

### 1. 📝 [문서 표준화 강령] 모든 문서의 헤더 필수 명시
- **규칙**: 프로젝트 내에서 작성 및 수정되는 모든 기술 문서, 스키마 설명서, 대응 일지(`.md` 파일) 상단에는 무조건 **[제목, 버전, 일시]** 정보가 명확히 머리글(Header)로 기재되어야 한다. 이를 통해 세션 변화에 관계없이 문서의 최신성과 컨텍스트를 즉각 추적할 수 있도록 한다.

### 2. 🛡️ [사전 안전 강령] 파일 삭제 및 일괄 정리 집행 전 사전 승인
- **규칙**: 프로젝트 내의 불필요한 임시 파일, 오래된 소스코드, spec 설정 등을 영구 삭제하거나 격리 청소(Declutter) 작업을 수행하기 전에는 **절대 자의적으로 먼저 실행하지 않는다.** 반드시 파일 전수조사 및 분류를 선행하고, 구체적인 정돈 플랜을 요약 보고하여 **대표님의 명확한 동의 및 승인**을 얻은 뒤 안전하게 정리를 집행한다.

### 3. 📂 [이슈 일지 보존 강령] 1회성 help.md 대응 일지의 날짜 병합 아카이빙
- **규칙**: 개발 도중 발생하는 임시 이슈나 세션 간 인수인계 지침은 루트 폴더 내에 일회성 대응 일지인 `help.md`로 밀접하게 소통 및 기록한다. 단, 세션이 최종적으로 매듭지어지거나 정리될 때는 해당 파일의 흔적을 루트에서 지우는 대신, **파일명에 해당 날짜를 결합(예: `help_YYYYMMDD.md`)하여 `/docs` 폴더 내로 이동 보존**함으로써 장기적인 트러블슈팅 히스토리를 자산화한다.

### 4. ⚙️ [배포 일관성 강령] 빌드와 릴리즈 포장의 파이프라인 단일화
- **규칙**: 릴리즈 준비 시 코드 컴파일(`build_exe.py`)과 데이터/계정 정보 정제 및 ZIP 패키징(`prepare_dist.py`)은 절대 수동으로 각각 실행하지 않는다. 휴먼 에러를 방지하기 위해 **단 하나의 진입점 실행만으로 빌드부터 클린 배포용 ZIP 압축까지 원스톱으로 처리되는 일관된 일괄 작업 파이프라인**을 준수하고 영구 보존한다.
